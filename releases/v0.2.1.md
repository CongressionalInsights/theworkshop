# TheWorkshop `v0.2.1` - Lessons-Driven Execution Quality + Investigative Scaffolds

TheWorkshop `v0.2.1` focuses on closing the gap between artifact completion and task-specific rigor.

It adds deterministic lessons application, investigation-oriented job scaffolds, and stricter content-quality gates so completion behavior aligns with real execution quality.

## What Is Included

### Lessons auto-application and ranking upgrades
- Added `scripts/lessons_apply.py` and `theworkshop lessons-apply`.
- `job_start.py` now auto-applies relevant lessons by default before transitioning to `in_progress`.
- New job-start controls:
  - `--no-apply-lessons`
  - `--lessons-limit`
  - `--lessons-include-global`
- Improved lessons ranking in `scripts/lessons_query.py` with deterministic scoring:
  - text similarity
  - tag overlap
  - linked ID overlap
  - recency and stable tie-breakers

### Additive lessons index compatibility
- `scripts/lessons_capture.py` now emits additive fields in `notes/lessons-index.json`:
  - `captured_at`
  - `context`
  - `worked`
  - `failed`
  - `recommendation`
- Older index entries remain compatible.

### Investigative job profiles
- Added `--job-profile` to `scripts/job_add.py`:
  - `default`
  - `investigation_attribution`
  - `identity_resolution`
- Non-default profiles prefill stronger outputs, acceptance criteria, and verification structure for attribution and entity-resolution workflows.

### Content-quality gate hardening
- `scripts/reward_eval.py` now computes specificity diagnostics and quality-sensitive next actions.
- `scripts/plan_check.py` now enforces lifecycle-aware quality policy:
  - `planned`: warnings
  - `in_progress` / `done`: hard failures for weak/placeholder content

### CI and regression coverage
- CI now includes:
  - `scripts/job_profile_test.py`
  - `scripts/lessons_apply_test.py`
  - `scripts/plan_check_content_quality_test.py`
- Added dedicated regression tests for profile scaffolding, lessons apply behavior, and strict quality gate behavior.

## Upgrade Impact

- Operator workflows using `job_start.py` now get automatic lessons insertion unless explicitly disabled.
- Teams relying on minimal boilerplate in active/done jobs may see `plan_check.py` failures until sections are made task-specific.
- Existing project artifacts remain valid under additive compatibility rules.

## Known Limitations

- Quality checks are deterministic but heuristic; they are designed to favor explicit, verifiable content rather than narrative brevity.
- Lessons relevance quality still depends on the quality of captured lesson entries.
