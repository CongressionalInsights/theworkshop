# TheWorkshop `v0.2.0` - Control Plane Reliability + Delegation + Readability

TheWorkshop `v0.2.0` publishes the current control-plane baseline for open-source use in Codex/Claude Code: stronger lifecycle correctness, executable orchestration, schema hardening, dashboard telemetry upgrades, and improved operator readability.

## What Is Included

### Lifecycle and state model
- Canonical transition engine (`transition.py`) across project/workstream/job state changes.
- First-class close semantics (`done` and `cancelled`) and alias surface in `scripts/theworkshop`.
- Rollup and consistency checks strengthened in lifecycle scripts and plan checks.

### Truth and completion integrity
- TruthGate fail-closed behavior integrated into completion pathways.
- Dependency freshness and stale invalidation controls (`input_snapshot.py`, `invalidate_downstream.py`).
- Completion semantics enforce multi-gate correctness (agreement + dependency/freshness + truth + reward).

### Orchestration and delegation
- Executable orchestration dispatch (`dispatch_orchestration.py`) from planned groups.
- Role profile resolution (`resolve_agent_profile.py`) and optional council planning (`council_plan.py`).
- Schema validation for machine artifacts (`schema_validate.py`) with shipped JSON schemas.

### Dashboard and monitoring
- Single-writer dashboard projector model and monitor runtime controls.
- Billing-aware spend display (`subscription_auth|metered_api|unknown`) with API-equivalent estimates.
- Human-first event readability with raw details drawers for machine payloads.
- Canonical subagent telemetry from `logs/agents.jsonl`, with dispatch compatibility telemetry retained.

### Image and credential reliability
- Image generation runner hardened for OSS environments.
- Env-first credentialing (`THEWORKSHOP_IMAGEGEN_API_KEY`) plus compatibility aliases.
- Optional keychain compatibility path retained and documented.

### Test and quality surface
- Expanded script-based regression coverage across lifecycle, truth/reward, orchestration, dashboard, billing, and credential paths.
- Added CI workflow to run syntax + critical regression suite on push/PR.

## Known Limitations

- Some tests rely on local tool/browser availability and may skip or degrade in constrained environments.
- Per-work-item spend attribution is allocation-based and approximate, not invoice-grade billing truth.
- Dispatch compatibility telemetry is retained for backward compatibility and may not be populated in manual-only flows.
- Council planning quality depends on available local model tooling and credentials.

## Install / Update

```bash
git clone https://github.com/CongressionalInsights/theworkshop.git
mkdir -p "$CODEX_HOME/skills"
cp -R theworkshop "$CODEX_HOME/skills/theworkshop"

# later updates
cd "$CODEX_HOME/skills/theworkshop" && git pull origin main
```

## First-Run Verification

```bash
python3 scripts/doctor.py
python3 scripts/project_new.py --name "v0.2.0 smoke"
python3 scripts/plan_check.py --project /path/to/project
python3 scripts/dashboard_build.py --project /path/to/project
python3 scripts/schema_validate.py --project /path/to/project
```
