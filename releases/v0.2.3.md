# TheWorkshop `v0.2.3` - OSS Baseline Hardening + Skill-Surface Autoresearch

TheWorkshop `v0.2.3` expands the public OSS baseline in two directions: it makes the repo more explicit about its portable/public contract, and it adds a repo-local outer-loop path for improving the skill surface without turning the whole repository into an autonomous search space.

It focuses on:
- a clearer OSS-vs-private boundary,
- a stronger delegated-work contract for telemetry, staged learning, and context locks,
- a reproducible autoresearch loop for tuning docs and repo-local agent guidance,
- refreshed public subagent/docs assets that match the current runtime model.

## What Is Included

### Repo-local skill autoresearch harness
- Added:
  - `scripts/skill_autoresearch_eval.py`
  - `scripts/skill_surface_contract_score.py`
  - `scripts/skill_autoresearch_eval_test.py`
  - `autoresearch/program.md`
  - `autoresearch/benchmark-pack.fast.json`
  - `autoresearch/benchmark-pack.full.json`
- The mutable surface is intentionally limited to:
  - `README.md`
  - `SKILL.md`
  - `references/prompting.md`
  - `references/workflow.md`
  - `references/templates.md`
  - `.codex/agents/*.toml`
- The scorer now adds partial-credit headroom for operator-facing contract quality instead of relying only on binary pass/fail checks.

### Stronger delegated-work contract
- Repo-local agent instructions and operator references now reinforce:
  - job-plan / verification-path grounding for delegated roles,
  - durable blocker evidence instead of hidden status narration,
  - truthful manual/external delegation telemetry through `theworkshop agent-log` and exactly-once `theworkshop agent-closeout`,
  - staged learning with curator-only promotion of durable memory and canonical lessons,
  - context-lock propagation so delegated and looped work reopen `context_ref`, honor `locked_decisions`, and keep `deferred_ideas` out of scope until refreshed.

### OSS baseline + docs refresh
- Reframed the repository as the **public OSS baseline** for TheWorkshop, separate from any private/custom local overlays.
- Added a profile-aware preflight surface with:
  - `python3 scripts/doctor.py --profile codex`
  - `python3 scripts/doctor.py --profile portable`
- Updated public docs to describe:
  - optional adapters vs portable core behavior,
  - native Codex subagents as the default delegation runtime,
  - current telemetry, staged-learning, and context-lock semantics.
- Added a public explainer page at `docs/subagents.html` and refreshed generated docs assets to match the current runtime model.

## Compatibility and Upgrade Impact

- Existing projects remain compatible; this release mostly tightens documentation, prompt contracts, and portable packaging expectations.
- The autoresearch harness is optional and repo-local; it does not change ordinary TheWorkshop runtime usage unless you choose to run it.
- Delegated and looped work now has clearer expectations around context locks and staged learning promotion, but these changes are additive clarifications of the current runtime model rather than a breaking format change.
