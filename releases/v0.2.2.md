# TheWorkshop `v0.2.2` - Monitor Policy Correctness + Strict Completion Evidence

TheWorkshop `v0.2.2` hardens OSS operator behavior where the previous defaults could permit ambiguous completion or unexpected dashboard policy persistence.

It focuses on:
- predictable monitor policy semantics,
- stricter evidence requirements for newly created jobs,
- aligned truth/plan/reward signals for missing execution logs and missing linked lessons.

## What Is Included

### Monitor policy semantics corrected
- `job_start.py --no-open` is now runtime-only and does not persist `monitor_open_policy`.
- Added explicit persistent policy override:
  - `job_start.py --monitor-policy always|once|manual`
- Documentation now clearly separates ephemeral runtime flags from persistent project policy.

### Strict completion evidence defaults for new jobs
- New jobs now default to requiring both:
  - WI execution evidence (`logs/execution.jsonl` entries), and
  - linked substantive lesson capture (`notes/lessons-index.json`).
- New frontmatter defaults:
  - `execution_log_required: true`
  - `execution_log_exemption_reason: ""`
  - `lesson_capture_required: true`
  - `lesson_capture_exemption_reason: ""`

### New truth checks
- Added:
  - `work_item_execution_logged`
  - `linked_lesson_captured`
- `scripts/truth_eval.py` now fails strict jobs when required evidence is missing, unless explicit exemption rationale is recorded.

### Plan and reward gate alignment
- `scripts/plan_check.py` now blocks `done` status for strict jobs missing required execution evidence or linked lessons.
- `scripts/reward_eval.py` now emits deterministic next actions for:
  - missing required execution logs,
  - missing required linked lesson capture.

## Compatibility and Upgrade Impact

- Existing jobs remain compatible:
  - jobs without the new strict fields are treated as non-strict unless explicitly opted in.
- Newly created jobs get strict-by-default behavior and can still use explicit exemptions with rationale when requirements are intentionally not applicable.
